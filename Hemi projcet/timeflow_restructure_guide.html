Backend request lifecycle
Router

Declares routes and chains middleware. No logic — delegates immediately to the controller.

auth.router.ts tasks.router.ts
Controller

Thin HTTP layer — extract params, call service, send response. No DB or business logic here.

auth.controller.ts tasks.controller.ts
Service

All business rules live here. Orchestrates repositories. Throws domain errors, not HTTP errors.

auth.service.ts tasks.service.ts
Repository

Only Prisma calls. Returns domain objects. Swapping the ORM means touching only this layer.

tasks.repository.ts habits.repository.ts
Frontend data lifecycle
Page component

Calls React Query hooks, passes data to dumb components. Handles loading/error states only.

Query hooks

useTasks.ts, useHabits.ts — wrap React Query. No fetch logic inside components.

Service functions

Pure async functions calling the API via axios. No state, no hooks — just returns data.

Axios instance

Centralized interceptors for auth token injection and automatic token refresh on 401.

import { NextResponse } from 'next/server'; import type { NextRequest } from 'next/server'; const PUBLIC_ROUTES = ['/login', '/register', '/forgot-password']; export function middleware(req: NextRequest) { const token = req.cookies.get('accessToken')?.value; const isPublic = PUBLIC_ROUTES.some(r => req.nextUrl.pathname.startsWith(r)); if (!token && !isPublic) { return NextResponse.redirect(new URL('/login', req.url)); } if (token && isPublic) { return NextResponse.redirect(new URL('/dashboard', req.url)); } return NextResponse.next(); } export const config = { matcher: ['/((?!api|_next|.*\\..*).*)'] };
The 12 restructuring rules
01
Separation of concerns
Router → Controller → Service → Repository. Each layer has one job and knows nothing about the layers below it except through interfaces.
02
Thin controllers
Controllers extract request data and call services. If a controller has an if statement about business logic, it belongs in the service.
03
Custom error hierarchy
Throw NotFoundError or ForbiddenError from services. The centralized error handler maps them to HTTP status codes.
04
Repository pattern
All Prisma calls live in *.repository.ts. Switching databases means touching only those files — nothing else changes.
05
Barrel exports
Every folder has an index.ts that re-exports everything. Imports become from '@/services' instead of deep relative paths.
06
Path aliases
Configure @/ in both tsconfig.json files. No more ../../../ — imports are readable and refactor-safe.
07
Validated env vars
Parse and validate all env vars at startup in config/env.ts using Zod. Fail fast with a clear message rather than a cryptic runtime error.
08
Route-scoped components
Components only used on one route live in app/(app)/tasks/components/. Shared components live in components/shared/.
09
React Query hooks
No fetch or axios calls inside components. Extract to useTasks.ts, useHabits.ts. Components only call hooks and render data.
10
Frontend service layer
services/tasks.service.ts holds all API calls. Hooks call services. If the API URL changes, you update one file — not a dozen hooks.
11
Zod-inferred types
Use z.infer<typeof schema> to derive TypeScript types. Validation schema and type definition are always in sync — one source of truth.
12
Next.js middleware auth
Replace client-side redirect logic with middleware.ts. Route protection happens at the edge — users never see a flash of protected content.

Step-by-step migration
1
Add path aliases
Configure @/* in both tsconfig.json files. For Next.js, also add paths to next.config.js. Run the project to confirm no import errors.
2
Create common/errors.ts
Write the error class hierarchy. Update errorHandler.ts to check instanceof AppError first, then Prisma errors, then default 500.
3
Extract repositories (one module at a time)
Start with tasks. Move all prisma.task.* calls from the controller into tasks.repository.ts. Confirm tests pass before moving on.
4
Create service layer
Move business logic from controllers to tasks.service.ts. Controller becomes: extract params → call service → send response. Replace res.status(500) with throw new AppError(...).
5
Add barrel exports
Create index.ts in every folder that re-exports its public surface. Update all imports to use the short alias form.
6
Create frontend services + hooks
Move axios calls from components into services/*.service.ts. Create React Query hooks in hooks/. Refactor page components to call hooks only.
7
Add Next.js middleware.ts
Replace client-side auth redirect logic with a root middleware.ts. Remove useEffect redirect guards from layout components.
8
Move tokens to httpOnly cookies
Store JWT tokens in httpOnly cookies (set by the backend) instead of localStorage. Eliminates XSS token theft risk. Update middleware.ts to read cookies instead.
Incremental approach: Migrate one module at a time (start with tasks). Each module can be restructured independently — you don't need to change everything at once. The architecture is stable as soon as the contracts between layers (function signatures) are defined.
